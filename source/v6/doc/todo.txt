Features:

On spare time:
- magnifier tool
- save geo on disk
- save build on disk

Bugs:
- driver crash while exploring after baking
- camera motion latency/inertia. This is particularly obvious when rendering in 1024x1024 at 30 fps.
- static colored dots artifacts
- check memory allocation, there is a mismatch between allocated and used data 

Optmization done:
- measure performance before optimization:
- compress colors: done
- culling: separe block pos from remaining data, pack all when culling
- culling: do the InterlockedMax in a post-compute (generalize the idea for all shaders)
- reduce sampling count: 5x using 3 samples
- encode BlockCellItem.nextID on 10 bits: 2 bits for a page and 8 bits for an index, this sould will keep write speed while increasing read speed
- encode BlockCellItem on 64 bits: depth 21bits, next 11bits, rgb 24bits, hitMask 8 bits
- 8 buffers of 4 cellItems: improve locality and remove interlocked exchange

Optmization aborted:
- precompute inverse of ray direction
- 4 stages: culling, decompression, trace, blend

Optmization plan:
- do an optmized version with plain data
- culling: cull the center and not the 8 vertices
- detect high AA pixels
- detect plain data
- skip min/max for known ray directions
- isolate each stage timings
- sort cells by distance
- appendconsume instead of interlockedadd
- sort by mip and precompute grid/cell scales/sizes
- remove AllMemoryBarrierWithGroupSync() when <= 64 threads
- take advantage of the silhouette of the box
